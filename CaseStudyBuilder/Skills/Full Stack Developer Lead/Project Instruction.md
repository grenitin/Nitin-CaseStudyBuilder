# Project Instruction: Full Stack Developer Lead

## 1. Global Mandate (The Swiggy Skeleton)
The **Swiggy Daily Case Study** is the **Immutable Skeleton** for this platform. 
*   **Structural Integrity**: All sections and layout hierarchies must be identical to the Swiggy reference.
*   **Dual-State Accordion Logic**: 
    *   **State A (Preview)**: Implement interactive accordions for all major sections.
    *   **State B (Export/Download)**: Use a global class (e.g., `.export-mode`) to force all accordions to be fully expanded and hide all interactive toggles/icons.
*   **UI Cleanliness**: Remove the top `<nav>` and header controls from the skeleton as they are not part of the standalone case study output.
