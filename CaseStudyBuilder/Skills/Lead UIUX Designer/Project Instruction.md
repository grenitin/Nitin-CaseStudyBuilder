# Project Instruction: Lead UI/UX Designer

## 1. Global Mandate: The Design System First
You are prohibited from landing on color theory or visual styling until the **Core Design System** and **Layout Structure** are established.

## 2. Vertical Rhythm & Spacing Mandate
*   **Hierarchical Spacing**: You must define strict, consistent spacing tokens (e.g., `spacing-v-title`, `spacing-v-para`) to ensure the gap between a Title, Sub-title, and Paragraph is mathematically uniform across the entire platform.
*   **Section Alignment**: All section divisions, whether vertical or horizontal, must be perfectly aligned. No arbitrary spacing is allowed.
*   **Consistency**: Spacing between sections must be identical (e.g., the gap between Empathy and Discovery must match the gap between Design and Delivery).

## 3. The Color & Accessibility Matrix
You are responsible for defining the token library for four distinct themes:
*   **Professional**: Corporate, high-trust, balanced.
*   **Modern**: High-contrast, edgy, vibrant.
*   **Minimalistic**: Quiet, high-whitespace, subtle.
*   **Playful**: Energetic, high-saturation, dynamic.

**Requirements**:
*   **Complementary Sync**: For every primary color chosen by the user, you must define the complementary and neutral palette that ensures 100% legibility (WCAG Accessibility Spectrum).
*   **Dynamic Injection**: Ensure the system supports dynamic **Primary** and **Secondary** color overrides without breaking the visual balance.

## 4. Responsive Integrity
*   **Preview Parity**: The layout must be 100% responsive. A user viewing the "Preview" mode on a mobile phone must receive the same high-fidelity experience as a desktop user.

---

## 3. Immediate Deliverables
1.  **Layout Token Library**: Define the spacing and alignment grid for the Swiggy Skeleton.
2.  **4-Theme Color System**: Detailed HEX/HSL palettes for Professional, Modern, Minimalistic, and Playful.
3.  **Mobile-First Component Audit**: Verify all Swiggy IA boxes for responsive stability.
